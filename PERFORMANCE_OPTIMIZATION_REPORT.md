# تقرير تحسين أداء الشارت وعرض الأسعار

## المشاكل المحددة التي تم العثور عليها

### 1. مهلة EODHD طويلة جداً (المشكلة الرئيسية)
**الملف:** `marketdata.php` (السطر 146-148)
- **المشكلة:** مهلة الاتصال بـ EODHD كانت 8 ثواني (max)
- **التأثير:** عندما يتأخر API أو يكون بطيئاً، ينتظر PHP حتى 8 ثواني لكل طلب
- **الحل:** تم تقليل المهلة إلى 4 ثواني (min بدلاً من max)

### 2. فترة التخزين المؤقت قصيرة جداً
**الملفات المتأثرة:**
- `api/lib/quotes.php` - EODHD_PRICE_TTL كانت 1-2 ثانية
- `api/lib/quote_cache_policy.php` - كانت قيم TTL منخفضة
- `api/lib/price_core.php` - eodhd_ttl كانت 1 ثانية
- `api/quote_hub.php` - eodhd_ttl كانت 1-2 ثانية
- `api/quote_active.php` - eodhd_ttl كانت 1 ثانية

**التأثير:** النظام يطلب الأسعار من API كل ثانية، مما يسبب:
- طلبات متكررة للـ API
- استهلاك سرعة الاستجابة
- احتمالية الحظر من مزودي الخدمة

**الحل:** تم رفع EODHD TTL إلى 3-5 ثواني

### 3. طلبات الـ watchlist المباشرة للأصول غير الكريبتو
**الملف:** `api/quotes.php` (السطر 324-330)
- **المشكلة:** `QUOTES_WATCHLIST_LIVE_NONCRYPTO=1` يسبب طلبات مباشرة للـ API
- **التأثير:** بطء كبير في الصفحة الرئيسية وقائمة المتابعة
- **الحل:** تم تعطيل هذه الميزة افتراضياً (تغيير من '1' إلى '0')

### 4. ميزانية الوقت للرسوم البيانية طويلة
**الملف:** `api/trade/candles.php` (السطر 39-45)
- **المشكلة:** CANDLES_REQUEST_BUDGET_MS كانت 6500 ملي ثانية
- **التأثير:** الرسوم البيانية تنتظر طويلاً قبل إرجاع النتيجة
- **الحل:** تم تقليل الميزانية إلى 3000 ملي ثانية

## ملف env.optimized

تم إنشاء ملف `env.optimized` يحتوي على جميع المتغيرات المحسنة التي يمكن نسخها إلى ملف البيئة الخاص بك.

## توصيات إضافية

### 1. تفعيل التخزين المؤقت أولاً للأصول غير الكريبتو
```
QUOTES_FOCUS_CACHE_FIRST_NONCRYPTO=1
```
هذا سيؤثر على طلبات التركيز (focus) لتظهر الأسعار من التخزين المؤقت أولاً.

### 2. تقليل معدل الطلبات
```
UPSTREAM_RATE_PER_SEC=10
UPSTREAM_BURST=20
```
لتجنب الحظر من مزودي الخدمة.

### 3. تمكين الاحتياطي من الشمعات
```
EODHD_ALLOW_CANDLE_QUOTE_FALLBACK=1
```
لاستخدام الشمعات كاحتياطي عند عدم توفر السعر المباشر.

## كيفية تطبيق التحسينات

1. انسخ محتوى ملف `env.optimized` إلى ملف البيئة الخاص بك (عادة `.env` أو `.env.railway`)
2. أعد تشغيل الخادم
3. اختبر الشارت والأسعار

## ملاحظات مهمة

- هذه التحسينات لا تؤثر على تصميم الواجهة أو مظهر الأسعار
- تم الحفاظ على جميع الوظائف الأصلية
- الأسعار ستظل محدثة لكن بتردد أقل (3-5 ثواني بدلاً من 1 ثانية)
- هذا يقلل الضغط على الخادم ويحسن تجربة المستخدم
